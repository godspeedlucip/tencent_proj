# 实习能量站 — 项目说明

## 1. 项目定位

实习能量站是一个 AI 驱动的实习生成长导航系统，覆盖实习生、导师、HR、招聘四个角色，将依赖个人经验的培养过程转化为可追踪、可预警、可复盘的协同流程。

## 2. 问题诊断

**实习生**入职后不知道学什么、做到什么程度算合格、如何证明成长，缺少清晰路径，不敢频繁打扰导师，容易焦虑。

**导师**面临带教是"附加任务"的困境——缺标准化工具，手动写反馈耗时，1:1 时记不住实习生做了什么，发现问题往往已错过最佳干预窗口。

**HR** 的信息散落在微信、飞书、口头反馈中，只能靠主观判断谁需要关注，做周报需手动拼凑，处于"被动救火"状态。

**招聘**在转正评估时几乎只依赖导师几句主观评价，看不到实际产出和横向对比，难以形成结构化留用判断。

## 3. 方案设计

**核心闭环：** 实习生每周提交 Check-in（含情绪胶囊）→ AI 分析成长状态并生成建议摘要 → 导师获得 1:1 沟通提纲，1 分钟内完成反馈 → HR 获得全局风险看板，异常时主动 push 通知 → 招聘获得多维度适岗报告。

**权限模型：** 实习生仅见自己的任务与建议；导师仅见所带教实习生；HR 见全局去敏数据；招聘仅见带人工复核的适岗结论。权限在后端 enforce。

**Human-in-the-loop：** AI 定位为"副驾"——可做摘要、分类、建议生成、风险解释，但不可直接给出淘汰结论。高风险、低适配结论强制附带人工复核备注，AI 与导师冲突时以导师为准。

**隐私红线：** 不读取私聊、不监控设备，仅分析系统内主动提交的数据。首次登录强制展示隐私声明。

## 4. AI 工具选型

**模型接入：** 选用 OpenAI SDK 兼容接口而非绑定单一模型，可对接 OpenAI、Azure、智谱、通义千问等，避免供应商锁定。

**架构取舍：** 采用 prompt + completion 直接调用，不引入 LangChain/LlamaIndex。当前场景的摘要、分类、建议生成均为单轮 prompt 即可完成，编排框架会增加延迟和复杂度。

**降级兜底：** API 调用设 5 秒超时，超时自动降级为预置 fallback JSON，确保 AI 服务不可用时 Demo 仍可完整演示。

## 5. 关键配置

**全栈架构：** 后端 FastAPI + SQLAlchemy + SQLite，前端 React 18 + TypeScript + Vite + Ant Design 5。SQLite 零配置单文件，SQLAlchemy 支持后续迁移至 PostgreSQL。

**角色隔离：** 权限判断在 API 层通过依赖注入 enforce，前端做 UI 差异化渲染。JWT 认证使用 sessionStorage 存储 token。

## 6. 迭代记录与效果评估

**迭代五阶段：**
1. 基础搭建——用户模型、JWT 认证、任务生命周期 API
2. 实习生模块——Check-in 升级、情绪胶囊、任务模板、AI 评审草稿
3. 导师模块——带教看板、基线评估、任务审核、周报与评分弹窗
4. HR 模块——导航布局、实习生/导师管理、风险指标
5. 收尾修复——权限修正、边界处理、baseline_scores 补全

**验收达成：** 四角色闭环完整；AI 超时 5 秒自动降级，核心流程不中断；首次登录 100% 展示隐私声明；所有 AI 负面判断均附带人工复核入口。
