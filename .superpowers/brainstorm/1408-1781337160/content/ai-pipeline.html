Section 1: AI Pipeline Architecture

Current → Proposed: From fake AI to real, context-aware, async AI

Current: Synchronous with no context

Problems:

  • No API key → always falls back
  • Only passes intern name, no real data
  • Blocks request (sync call)
  • No post-Check-in analysis
  • No confidence scores
  • Risk detection never called
Proposed: Async + Context-Rich + Multi-Provider

Changes:

  • OpenAI-compatible providers (DeepSeek/GPT)
  • Rich context: tasks + checkins + feedback history
  • FastAPI BackgroundTasks for analysis
  • Auto-triggered after Check-in submit
  • Confidence + evidence on every output
  • Risk detection integrated into flow

Post-Check-in AI Flow

Async Pipeline: Check-in → Analysis → Multi-Output
1. Submit
Intern submits Check-in
→ Returns 202 Accepted
→ "分析中，稍后刷新"
2. Analyze
BackgroundTask runs:
Text → growth keywords
Sentiment → risk signals
Compare → trend detection
3. Generate
Save risk_signal records
Update intern growth summary
Generate mentor talking points
Refresh daily tip context
4. Consume
HR sees new risk alerts
Mentor gets updated outline
Intern sees growth keywords
All with confidence scores

LLM Provider Strategy

Task Model Timeout Fallback
Daily Tip Cheap (DeepSeek-V3 / GPT-4o-mini) 3s Role-aware pre-baked tips
Talking Points (1:1 outline) Balanced (DeepSeek-R1 / GPT-4o) 5s Template outlines by intern status
Check-in Analysis (keywords + risk) Cheap (batch mode) 8s Keyword extraction rules
HR Weekly Report Best (DeepSeek-R1 / GPT-4o) 10s Statistical summary + rule-based actions
Fit Report Balanced 8s Score-based tier assignment

Key Design Decisions

A

Provider: DeepSeek (via OpenAI-compatible API)

Cheaper than GPT-4o, good Chinese NLP for the 中文 Check-in content. Same API shape as OpenAI = zero code change.

B

Provider: OpenAI GPT-4o-mini + GPT-4o

Better JSON structured output reliability. Higher cost but more consistent formatting for reports.
